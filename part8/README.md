### git 

- commit :: 로컬에 그시점을 저장하는 개념
- push :: 원격저장소에 저장하는 개념 ( 로컬과 원격저장소에 같은 내용을 저장)
- pull :: 원격저장소에 있는걸 받는것
- fetch :: 변경사항을 받는것

##### Git branch 

- master :: 운영을 위한 브랜치 개념 ->최종적으로 운영 서버영
- develop  - 2차 브랜치 만들어서 한다는 설명 -> develop 브랜치는 개발서버(테스트)

##### node nodemon

`nodemon`  npx nodemon '서버명' :: 자동서버

`npx` :: 파일 설치하기는 싫고 일회성으로 사용
